## 5. 選定方向（已轉向）

**領域：雙北買房決策輔助平台**
**技術重心：Lambda 架構（batch 為主 + stream 警示）**

### 專案：房價溫度計 (HouseTherm)
> 用「租金」當錨點，幫雙北自住買房族判斷：現在房價跟過去比偏高還偏低、
> 用新青安買要付多少、以及有哪些領先指標顯示未來下修風險。

### 核心分析主線：房價租金比 (Price-to-Rent Ratio, PRR)

```
PRR = 每坪房價 / 每坪年租金        租金收益率 = 1 / PRR
```

- 雙北 PRR 出名地高（收益率常僅 1.5%–2.5%，PRR 約 40–60 倍）→ 量化「房價相對租金基本面偏貴」。
- **租金資料 = 估值的分母**：把開價換算成「幾年租金回本」，跟該區歷史、跟其他區比 → 看出貴不貴。

### 三大功能
1. **估值（貴不貴）**：PRR 偏離歷史均值 + 房價歷史百分位 + 房價所得比。
2. **時機風險（會不會跌）**：**透明風險訊號儀表板**（不做黑盒預測）。
3. **可負擔性（買得起嗎）**：新青安月付試算 + 貸款負擔率。

### 風險訊號儀表板（取代「預測」）
| 訊號 | 含義 | 資料源 |
|------|------|--------|
| PRR 偏離歷史均值 | 均值回歸壓力（偏貴→有下修空間） | 實價登錄 + 591 |
| 交易量趨勢（量先價行） | 量縮常領先價跌 | 實價登錄 |
| 法拍屋量趨勢 | 「融資斷頭」代理變數（被迫拋售↑=承壓） | 司法院法拍 |
| 房貸利率 + 央行信用管制 | 資金面鬆緊 | 央行公告 |
| 房價所得比 / 貸款負擔率 | 可負擔性 | 內政部不動產資訊平台 |
| 政策/新聞事件 | 新青安、囤房稅、平均地權條例 | 新聞爬蟲 |

> 報告定位：**本系統不預測價格，而是整合領先指標供買方評估時機風險**。誠實、有學理、好寫。

### 用詞釐清
- **「融資斷頭」**（房市）→ 用**法拍屋量上升**當代理指標（過度槓桿買方被迫拋售）。
- **「政府補救」** → 新青安政策調整、央行選擇性信用管制鬆緊，納入政策/新聞訊號。

### 目標客戶（雙邊市場）
- **房仲端（B2B，付費主力）**：以「信任/透明」差異化的房仲、買方代理、想對殺價型客戶建立信任的業務。用客觀數據證明「開價合理、我沒騙你」→ 加速成交、減少議價拉鋸。當 sales-enablement 工具。
- **購屋族端（B2C，引流）**：雙北首購 / 自住買房族，尤其考慮新青安的人。免費或 freemium，衝流量與口碑，反過來證明工具公信力。
- ⚠️ **利益衝突（須正面處理，Go-to-Market 素材）**：工具會說「偏貴」，可能不利成交，所以房仲客戶不是「所有房仲」而是「信任差異化」那群；要誠實討論為何有些房仲不想用、如何鎖定願意用的人。
- **變現**：房仲訂閱/席次制（產生給客戶看的透明估值報告）；購屋族免費引流 → 雙邊網路效應。
- 需求證據超好找（PTT home-sale、Dcard、新青安討論、房仲訪談），rubric 看重蒐集過程(25%)。

### 資料源（雙北，偏合法）
| 資料 | 來源 | 合法性 |
|------|------|--------|
| 成交價（每坪/時間/區域） | 內政部實價登錄（季度批次 CSV） | ✅ 開放資料 |
| 租金 | 591 租屋（輔） | ⚠️ ToS，誠實討論 |
| 法拍量 | 司法院法拍屋查詢 | ✅ 公開 |
| 房價所得比/貸款負擔率 | 內政部不動產資訊平台（季） | ✅ 開放資料 |
| 新青安條件/利率 | 財政部、八大行庫公告 | ✅ 公開 |
| 房市新聞/政策 | 新聞網房市關鍵字 | ⚠️ 限速、不重發原文 |

### 範圍（已決定：核心優先）
**Phase 1（先做，end-to-end 打通）**：實價登錄 + 591 → PRR 估值 + 房價歷史高低 + 新青安月付試算。
**Phase 2（行有餘力）**：法拍量趨勢 + 新聞/政策事件 stream 警示（展示完整 Lambda）。

### Lambda 架構
```
資料源(實價登錄/591) → Scrapers → Kafka(message queue)
   ├─ Batch Layer:  Spark batch → PRR 估值 / 房價歷史百分位 / 趨勢
   └─ Speed Layer:  Spark Streaming → 新聞/政策/利率事件警示 (Phase 2)
        ↓
   Serving: Redis(即時) + PostgreSQL(歷史/行情)
        ↓
   Delivery: Web Dashboard(房價溫度計) / 新青安試算 / REST API
```

## 6. 風險與注意（push back）

1. **不做價格預測**——改成透明風險訊號儀表板，誠實且不易被打臉。
2. **爬蟲合法性**——主資料源用合法的實價登錄、政府開放資料；591/新聞當輔助並誠實討論 ToS/PDPA（Go-to-Market 加分）。
3. **不要硬塞全部組件**——Docker compose 單機跑即可，重點是架構說清楚、能 demo；時間緊先做 Phase 1。
4. **B2C 付費意願**——報告需正面處理：用「省下的決策時間/避免買在高點的金額」量化價值。
