# 需求驗證方法論與發現 (Demand Validation)

> 對應 rubric **Component 2（需求證據與資料蒐集過程，25%）**。
> 評分要求「記錄完整過程，不只結論」+「提供可重現資料蒐集的腳本」，故此文件記錄方法、清理、假設與結果。

## 1. 研究問題

房價溫度計的核心假設：**雙北房價相對租金基本面偏貴，且目前接近歷史高點**——
若此假設成立，買方「現在該不該買、這個開價合不合理」的判斷需求才真實存在。

## 2. 資料來源（合法）

| 資料 | 來源 | 期間 | 原始筆數 |
|------|------|------|------|
| 不動產買賣成交 | 內政部實價登錄開放資料（季度批次） | 111S1–115S1 (17 季) | 344,626 |
| 不動產租賃成交 | 同上（租賃實價登錄） | 111S1–115S1 | 161,246 |

- 端點：`https://plvr.land.moi.gov.tw/DownloadSeason`（政府開放資料，可重新利用）。
- 縣市：台北市(a)、新北市(f)。下載腳本：[`../ingestion/scrapers/lvr_gov.py`](../ingestion/scrapers/lvr_gov.py)（含快取、未釋出季別自動略過）。

## 3. 指標定義

```
PRR (房價租金比) = 買賣單價(元/m²) / (租賃月單價(元/m²) × 12)
租金收益率        = 1 / PRR
```
（坪數換算在分子分母約掉，故可直接用每 m² 單價計算。）

## 4. 資料清理與假設

清理後：**買賣 228,767 筆 / 租賃(整層) 56,176 筆**。規則：

- **分房型**：依 `建物型態` 映射為 大樓 / 華廈 / 公寓 / 透天（套房雙北幾乎無登記、店面等非住宅排除）。
  - **為何**：混合房型會讓中位數失真（如老公寓 40 萬混入大樓 56 萬），所有指標一律「同房型內」比較。
- **排除車位灌水**：用 `(總價−車位總價) / (建物面積−車位面積)` 計算「純房屋每 m² 單價」，避免含車位成交壓低單價。
- **踢非市場交易**：`備註` 命中「親友／特殊關係／關係人／債務／債權／拍賣／破產」者剔除（價格偏離行情）。
- **買賣**：只留 `交易標的`「房地(含建物)」、`主要用途=住家用`、純房屋單價 > 0。
- **租賃**：`主要用途=住家用`、單價 > 0、`出租型態` 含「整」（整層/整棟），並**與買賣同房型配對**。
  - **關鍵假設**：用「整層」租賃對應「買整戶」，避免分租套房/雅房每 m² 灌水（會低估 PRR）。
- **去極端值（IQR 法）**：各 (城市×行政區×房型) 群組保留 `[Q1−1.5·IQR, Q3+1.5·IQR]` 後取中位數。
- 樣本門檻：買賣 < 30 或 租賃 < 15 的群組不計（避免小樣本雜訊）。

## 5. 發現

**① 房價相對租金偏貴（PRR 高 / 收益率低）— 以房型分列**
| 城市 | 房型 | 買賣中位 | PRR | 租金收益率 |
|------|------|---------|-----|-----------|
| 台北市 | 大樓 | 82.3 萬/坪 | 44.9 | 2.23% |
| 台北市 | 公寓 | 58.7 萬/坪 | 52.3 | 1.91% |
| 新北市 | 大樓 | 46.3 萬/坪 | 59.6 | 1.68% |
| 新北市 | 公寓 | 37.9 萬/坪 | 43.4 | 2.30% |

> 對照健康租金收益率約 4–5%，雙北僅 1.7–2.3%，需 43–60 年租金回本。
> 分房型揭露反差：台北大樓 PRR 反而低於公寓（新大樓租金撐得住），新北大樓 PRR 最高。

**② 行政區差異大**（大樓 PRR：南港 67 / 大安 63 ↔ 蛋白區 40 上下）→ 同為雙北貴的程度天差地遠，買方需分區分型情報。

**③ 接近歷史高點**：含 115S1（2026 Q1），台北最新季 86.9 萬/坪、新北 51.0 萬/坪，皆為 4 年區間最高、高於過去 94% 季別；2024 後明顯上揚。
